# PRD – AI·검색 노출 진단 리포트 생성 시스템 (ReachCheck MVP)

## 1. 목적 (Purpose)
본 PRD는  
**상호명 입력 → 가게 선택 → AI·검색·지도 노출 상태 분석 → PDF 리포트 생성**  
까지 자동으로 수행하는 시스템을 정의한다.

최종 산출물은 **Page 1~4로 구성된 PDF 리포트**이며,  
리포트의 문장·구조·표현은 모두 **자동 생성 가능**해야 한다.

---

## 2. 최종 산출물 정의 (Output Definition)

### 형식
- PDF (A4)
- 총 4페이지
- 각 페이지는 고정된 구조를 가지며, 내용만 동적으로 채워진다.

---

## 3. 리포트 페이지별 요구사항

---

## Page 1. AI·검색 노출 진단 요약

### 3.1 헤더
- 가게명
- 리포트 제목:  
  `[가게명] AI·검색 노출 진단 요약`
- 진단 기준일: 생성 시점 기준 YYYY.MM.DD

---

### 3.2 현재 상태 요약 (자동 생성 규칙)

#### 입력 데이터
- 지도 채널별 정보 정확도
- AI 엔진별 언급 여부
- 정보 일치도 결과

#### 생성 로직
- 지도 정확도 ≥ 70% → “일부 정확”
- AI 엔진 언급률 < 50% → “안정적 인식 실패”
- AI 엔진 간 설명 상이 → “일관된 인상 부족”

#### 출력 문장 템플릿
현재 이 가게는
지도 정보는 일부 정확하지만,
AI 추천 단계에서는 안정적으로 인식되지 않고 있습니다.

특히 AI 엔진별로 언급 여부와 설명 내용이 달라
고객에게 일관된 인상을 주지 못하고 있습니다.

---

### 3.3 지도 검색 채널 상태 (채널별)

#### 대상 채널
- 네이버
- 카카오
- 구글

#### 평가 기준
- 이름 / 주소 / 전화 / 영업시간 일치 여부

#### 상태 분류
- 🟢 정상
- 🟡 일부 누락 또는 불일치
- 🔴 명확한 오류

#### 출력 규칙
- 각 채널은 개별 블록으로 출력
- 상태 아이콘 + 한 줄 설명 자동 생성

---

### 3.4 AI 엔진 노출 상태 요약

#### 대상 AI
- ChatGPT
- Gemini
- Claude
- AI Overview / Perplexity

#### 측정 기준
- 기준 질문 n개 중 언급 횟수
- 언급 시 설명 포함 여부

#### 상태 분류
- 🟢 언급률 ≥ 60%
- 🟡 언급률 20~59%
- 🔴 언급률 < 20%

---

### 3.5 가게 정보 일치도

#### 비교 항목
- 가게명
- 주소
- 전화번호

#### 판정 로직
- 100% 동일 → “정보 일치”
- 일부 불일치 → “정보 일부 상이”

#### 설명 문장 자동 삽입
AI 검색에서 정보 불일치가 왜 문제가 되는지에 대한  
**고정 설명 블록**을 포함한다.

---

### 3.6 리스크 / 기회 Top3

#### 리스크 생성 기준
- AI 미언급 비율
- 지도 정보 오류
- 리뷰 부정 키워드 빈도

#### 기회 생성 기준
- 리뷰 긍정 키워드
- 프로그램/설명 구조화 가능성
- FAQ 확장 가능성

---

## Page 2. AI는 이렇게 말하고 있습니다

### 4.1 기준 질문 세트
- 지역 + 업종 추천 질문
- 가격 / 체험 / 예약 관련 질문
- 후기 기반 질문

※ 질문 세트는 고정, 응답만 동적

---

### 4.2 AI 엔진별 응답 분석

#### 각 AI 엔진별 출력 항목
- 언급 상태 (🟢🟡🔴)
- 실제 응답 요약 (1~2문장)
- 문제점
- 해석 (자동 해석 문장)

#### 해석 문장 규칙
- 미언급 → “추천 후보로 인식되지 않음”
- 이름만 언급 → “차별점 인식 부족”
- 설명 포함 → “리뷰/설명 자산 반영”

---

### 4.3 AI 인식 차이 요약 표
- 엔진별 언급 여부
- 설명 정확도
- 주요 문제

---

### 4.4 종합 해석 블록
AI 엔진마다 다른 정보 신호를 사용한다는  
**고정 설명 문단** 삽입

## Page 3. 무엇을 고쳐야 하나

고정 메시지 포함

## Page 4. 바로 써먹는 AI 문장 (맛보기)

#### 자동 생성 템플릿
[가게명]은 [지역]에서 [대상]을 위한 [업종/서비스]를 제공하며,
[강점1]과 [강점2]로 알려진 곳입니다.


---

### 6.2 문장 중요성 설명

### 6.3 전환 블록 (옵션)

- 개선 문장/FAQ/리뷰 응답 제공 안내

---

## 7. 자동 생성 원칙 (Critical Rules)

1. 모든 문장은 **데이터 + 템플릿 조합**으로 생성 가능해야 한다.
2. 판단 문장은 반드시 **근거 지표(비율/횟수)**를 기반으로 한다.
3. 캡처 실패 시에도 리포트는 생성된다.
4. 표현은 “분석 리포트 톤”을 유지한다.
5. 과장·보장·대행 표현은 금지한다.

---

## 8. 기술 전제 (Implementation Assumptions)
- place_id 기반 데이터 수집
- 지도 API + Playwright 캡처 혼합
- 규칙 기반 분석 + (선택) LLM 문장화
- HTML → PDF 렌더링

---

## 9. 성공 기준
- 리포트 구조가 항상 Page 1~4로 동일하게 출력
- 서로 다른 가게 입력 시도 **톤과 구조 일관성 유지**
- “자동 생성 리포트”임을 설명 없이도 이해 가능
